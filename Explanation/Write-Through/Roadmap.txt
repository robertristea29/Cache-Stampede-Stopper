PHASE 1: Create Write-Through POST Endpoint
File: index.ts

Task: Add a NEW POST endpoint (keep Write-Behind endpoint separate for comparison)

// NEW endpoint for Write-Through comparison
POST /matches/:id/goal/write-through
  1. Validate input (matchId, score_home, score_away)
  2. Update PostgreSQL database FIRST (synchronous)
     - Execute: UPDATE matches SET score_home=?, score_away=?, last_updated=NOW() WHERE id=?
     - If error → Return 500 immediately
  3. Only after DB success → Update Redis cache
     - Set cache with CACHE_TTL (30s)
  4. Return 200 OK with full match data

Key Difference from Write-Behind:
  ✅ Write-Behind:    Cache → Queue → Return | Worker updates DB later
  ✅ Write-Through:   DB → Cache → Return (user waits for DB)

Endpoint Response Time Comparison:
  Write-Behind:    ~5-10ms (cache + queue)
  Write-Through:  ~50-150ms (database operation)

  Implementation Checklist:

 Create /matches/:id/goal/write-through POST endpoint
 Fetch full match object first (from cache or DB)
 Execute database UPDATE synchronously
 On success, update Redis cache
 Return full match object
 Error handling if database fails
 Console logging for debugging
